# 浓缩邮件格式规范

## 邮件结构最佳实践

### 1. agentNote 结构模板

agentNote 是邮件中最重要的部分，它出现在文章列表之前，决定了用户是否继续阅读。

**标准格式：**

```
📊 今日信息浓缩（共处理 X 篇，精选 Y 篇）

🔥 核心发现（Top 3）
1. [最重要的发现] — 来自《文章标题》
2. [次重要的发现] — 来自《文章标题》
3. [第三重要的发现] — 综合多篇文章

📌 主题聚焦
• 🤖 AI/大模型：[1-2句核心信息]
• 💻 前端/工程化：[1-2句核心信息]
• 🔧 后端/架构：[1-2句核心信息]
（仅列出今天有内容的主题）

💡 今日行动建议
1. [具体建议] → 推荐阅读《文章标题》
2. [具体建议] → 关注 [技术/趋势]

📈 趋势信号
• [跨文章分析得出的趋势判断]
```

### 2. 写作原则

#### 信息密度优先
- 一句话 = 一个信息点，不要水
- "GPT-5 下周发布" > "OpenAI 近期可能会有新的大模型发布的消息"
- 用动词开头："关注..."、"尝试..."、"了解..."

#### 去重与合并
- 3 篇讲 React 19 的文章 → 合并为 1 条："React 19 发布，核心变化：编译器优化、Server Components 稳定化、新 Hooks"
- 不要重复列出同一主题的多篇文章摘要

#### 个性化标注
- 基于用户兴趣标签，对相关内容加 ⭐ 标记
- 如果某个主题匹配用户多个兴趣标签，优先级提升

### 3. 内容选择策略

| 内容数量 | 策略 |
|---------|------|
| 0 篇 | 不发送邮件 |
| 1-3 篇 | 全部包含，agentNote 中注明"今日内容较少" |
| 4-10 篇 | 全部包含，正常浓缩总结 |
| 11-20 篇 | 精选 Top 10，按评分 + 用户兴趣匹配度排序 |
| 20+ 篇 | 精选 Top 15，按主题分组择优 |

### 4. agentNote 字数控制

- **核心发现**：每条不超过 40 字
- **主题聚焦**：每个主题不超过 60 字
- **行动建议**：每条不超过 50 字
- **总计**：不超过 500 字

### 5. 与现有邮件模板的配合

本 Skill 使用 `send_daily_digest` 工具发送邮件。该工具已有精美的 HTML 邮件模板，会自动：
- 按评分排序展示文章
- 高分文章（≥60）完整展开（含摘要、评分拆解、行动建议）
- 低分文章折叠显示
- agentNote 以蓝色高亮区块展示在邮件顶部

因此，本 Skill 的 agentNote 相当于邮件的**编者按/导读**，是用户首先看到的内容。
要让它成为整封邮件中**信息密度最高、最值得阅读的部分**。

### 6. 特殊场景处理

#### 全部低分（均 < 50）
agentNote 开头注明："今日内容整体评分偏低，建议快速浏览标题即可。"

#### 单一主题集中
agentNote 突出该主题："今日 80% 的内容都聚焦在 [主题]，看来这是当前热点。"

#### 与用户兴趣不匹配
agentNote 说明："今日内容与您的核心兴趣匹配度一般，但发现了一些跨领域的有趣内容。"
